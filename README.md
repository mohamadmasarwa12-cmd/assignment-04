# Assignment 04 – Creating PDF Documents Using LaTeX

## Introduction

In this assignment, I explored the process of creating academic PDF documents using LaTeX and LuaLaTeX. Although I had previous experience with programming environments, this was my first practical exposure to professional document preparation using LaTeX.

The main objective of the assignment was to learn how to create a structured PDF document, organize project files correctly, and use AI-assisted development tools to generate and improve the required code. The assignment also emphasized documenting the development process through dedicated Markdown files and maintaining a well-organized GitHub repository.

---

## Project Description

The project focuses on creating an academic report that demonstrates the capabilities of LaTeX and LuaLaTeX. The final document includes structured sections, a table of contents, tables, figures, graphs, references, and a custom document class.

Unlike traditional word processors, LaTeX separates content from formatting. During this project, I learned how this approach simplifies the creation of consistent and professional-looking academic documents.

---

## Development Process

### Environment Setup

The first stage involved installing and configuring MiKTeX and verifying that LuaLaTeX was working correctly through PowerShell. After successful installation, a project structure was created according to the assignment requirements.

### Document Development

The document was developed section by section. The report includes background information about TeX and LaTeX, explanations of their advantages, examples of document structure, and demonstrations of tables, graphs, and figures.

### Adding Advanced Components

Additional components were integrated into the document, including:

* A comparison table between LaTeX and Microsoft Word.
* A graph generated using PGFPlots.
* The official LaTeX Project logo as a figure.
* References and bibliography.
* Automatic table of contents generation.

### Compilation and Testing

The document was compiled using LuaLaTeX. Several compilation attempts were performed to ensure that the table of contents, figures, graphs, and references appeared correctly in the final PDF output.

---

## Challenges Encountered

One of the main challenges was understanding the LaTeX compilation workflow. Unlike standard word processors, even small syntax errors can prevent successful PDF generation.

Another challenge involved correctly integrating images and external resources while maintaining a clean project structure. Through troubleshooting and multiple compilation cycles, these issues were resolved successfully.

## AI-Assisted Development and Prompt Engineering

A central part of this assignment was learning how to communicate effectively with AI tools in order to generate the required LaTeX code and supporting documentation. Rather than manually writing every component from scratch, the development process involved creating clear instructions and refining prompts to obtain accurate outputs.

Several prompts were used throughout the project. These prompts focused on generating the document structure, creating LaTeX sections, building tables and graphs, integrating images, producing Markdown documentation files, and troubleshooting compilation errors.

The AI-generated outputs were not accepted automatically. Each generated component was reviewed, tested, modified when necessary, and integrated into the final project structure. This iterative process improved both the quality of the final document and my understanding of LaTeX development.

The experience demonstrated that successful AI-assisted development depends not only on the capabilities of the AI system, but also on the quality and clarity of the instructions provided by the user. Effective prompt engineering significantly reduced development time while maintaining full control over the final deliverables.
